# 全局规则（项目级）

## 1. 项目目标

- 本项目用于维护通用语言学习刷题 skill：`lang-drill-coach`。
- 目标是让 OpenClaw、Codex、Claude Code 等智能体根据学习目标、考纲、学习背景、学习进度和复习算法，生成语言学习测验并维护长期复习状态。
- 当前优先适配日语与英语；日语保留大学日语四级 2023 考纲和高中日语 2020 课程资源，英语在用户确定考试目标后导入对应考纲。

## 2. 规则读取顺序

- 开始任何实质工作前，先读取本文件。
- 继续读取 `doc/项目局部规则.md`。
- 若全局规则与局部规则冲突，以局部规则为准；若局部规则未覆盖，则继续遵循全局规则。

## 3. Git 与撤销

- 在结构性改动、批量生成、批量删除、目录迁移前，先检查 Git 状态。
- 若当前仓库已有未提交改动，先创建检查点提交，再继续后续整理。
- 禁止擅自回退或覆盖用户已有改动。
- 发布到 GitHub 前必须确认历史中没有 `.env`、真实密钥、个人学习记录、私人日志或用户使用痕迹。

## 4. Skill 与程序

- 项目内 skill 的源码位置固定为 `skills/lang-drill-coach/`。
- 若需要同步到外部 skills 仓库，只能以项目内目录为源同步到 `D:\2Folder\skills\lang-drill-coach`。
- 能由脚本稳定处理的结构化任务，优先交给脚本执行；题目质量、讲解质量、考纲解释和教学判断由 agent 负责复核。
- 正式题目必须由 agent 基于考纲、真题题型和候选知识点编写，再落入 `data/study.db`；脚本只负责候选选择、持久化、判题回写和复习调度。

## 5. 初始化流程

- 第一次载入 skill 时，必须了解用户的目标语言、考试目标、学习背景、截止时间、每日题量或学习时长、个人偏好、当前掌握情况、后续录入词汇/语法的方式，以及是否需要提醒。
- 若考纲未入库，先询问导入本地资料还是主动搜索官方/可靠公开资料；考纲文件和结构化数据必须标注年份。
- 导入考纲后，明确后续题型范围和题型比例；询问是否需要收集或导入近几年真题作为参考。
- 对已经学过但缺少准确学习日期的内容，采用保守复习计划，不把它们视为完全掌握。

## 6. 记录与隐私

- 每完成一个阶段性任务，更新 `doc/进展记录.md`；普通问答、网络搜索等不涉及项目文件修改的内容不需要写入。
- 每条进展记录必须包含精确到分钟的时间段：`YYYY-MM-DD HH:mm ~ YYYY-MM-DD HH:mm`。
- `.env` 是真实环境变量账本，必须加入 `.gitignore` 且禁止提交。
- `.env.example` 是假账本，只能包含变量名、占位值、示例值和必要说明。
- 禁止把真实密钥、token、cookie、数据库密码、私有地址写入 `.env.example`、README、进展记录或聊天可复制代码块。

## 7. 资料与测试

- 测试文件统一放在 `try/`，保证可清空且不影响项目。
- `data/kb/material-inbox/` 用于放待导入的考纲、真题、教材或词表原始资料。
- 历年真题默认先建立索引和导入口，不把来源不明的完整试题全文作为默认系统资产。
- 工具日志只用于调试，不是学习数据事实来源；正式学习状态以 `data/study.db` 为准。
