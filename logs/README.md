# 日志目录

本目录存放项目脚本的运行日志，主要用于排查 bug、回放工具调用和观察项目健康状态。

## 结构

- `tool_runs.jsonl`
  - 全量运行索引，每行一条 JSON 记录。
  - 记录工具名、参数、开始/结束时间、耗时、状态、错误类型、详情日志路径和最近输出摘要。
- `health_snapshot.json`
  - 最近 7 天的工具运行健康摘要。
  - 包含总运行次数、成功/失败次数、最近一次失败和各工具的运行/失败分布。
- `YYYY-MM-DD/*.log`
  - 单次运行的详情日志。
  - 记录命令参数、工作目录、输出摘要以及异常 traceback。

## 保留策略

- 每次工具运行时自动清理旧日志。
- 按天的详情日志默认保留 `30` 天。
- `tool_runs.jsonl` 索引默认保留 `60` 天的记录。

## 注意

- 本目录默认不纳入 Git 提交，只保留本说明和占位文件。
- 日志用于调试和健康巡检，不是学习数据的事实来源；正式学习状态仍以 `data/study.db` 为准。
