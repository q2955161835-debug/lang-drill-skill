# 项目局部规则

最后更新：2026-05-15 20:12

## 1. Skill 位置

- 项目内 `skills/lang-drill-coach/` 是通用语言刷题 skill 的源码与事实来源。
- 同步到外部 skills 目录时，只能从项目内目录发布到 `D:\2Folder\skills\lang-drill-coach`。

## 2. 初始化约束

- 新用户或新考试目标必须先建档，再生成正式题目。
- 建档信息至少包括：目标语言、考试目标、当前水平、已学内容、截止时间、每日题量或学习时长、偏好、弱项、提醒需求、词汇/语法录入方式。
- 考纲缺失时，先导入本地资料或搜索官方/可靠公开资料；结构化入库时必须保留考试名、年份、来源和 `source_scope`。
- 题型必须在考纲或真题参考明确后再固定；没有题型资料时只能先生成通用诊断题或小规模练习。

## 3. 组题约束

- 日常流程固定为：初始化 -> 导入新内容/考纲 -> 选择候选 -> agent 编写完整题单 -> 整套落库 -> 聊天逐题展示 -> 判题讲解 -> 即时回写状态。
- 聊天中一次只给一道题，不提前泄露答案；题头格式为“第 N 题 / 共 M 题”。
- 用户答题后，先给对错判定，再解释原因，再回顾相关词法，最后询问是否继续。
- 用户若回答“不会”“不确定”“跳过”等，默认按答错处理并即时回写。
- 一道题可以覆盖多个词汇、语法和句法点；在保证讲解清晰的前提下，优先提高单题信息密度。
- 选择题选项必须打散，答案不能长期固定在同一选项位。
- 阅读、听力、会话等材料题允许“一段材料 + 多题”，同一材料的子题必须连续存储。
- 题目唯一正式来源是 `data/study.db`；习题日记只作为已使用题目的可读回放。

## 4. 复习与进度

- 候选抽取优先级：当天新学 -> 到期复习 -> 复习中/已掌握内容 -> 错题回流 -> 考纲兜底。
- 缺少精确学习日期的既有内容应进入保守复习队列，不直接算作完全掌握。
- 每题作答后必须即时写入 attempts、题目状态、词汇/语法掌握度、错题记录和下次复习时间。
- 一整套正式会话完成后，运行会话级熟练度校准和当天数据审计。

## 5. 资料边界

- 日语 CJT4 2023、高中日语 2020 等已清理为可复用考纲资产，可以作为默认日语种子。
- 英语考试需要在用户确定目标后导入对应考纲，推荐按 `data/kb/<exam-id>/official_vocab_YYYY.json` 等命名。
- 真题用于题型和风格参考；不得把来源不明的完整试题全文当作默认发布资产。

## 6. 进展记录

- 一次对话一般只在阶段收口时更新一次 `doc/进展记录.md`。
- 记录内容保持简洁，包含完成内容、文件清单与用途说明；若出现错误，写入“错误汇报”。
